# Azraeel

I build systems for simulation, game design, AI-assisted workflows, and structured worldbuilding.

Most of my work sits at the intersection of strategy games, military simulation, procedural tools, and persistent fictional worlds. I focus on projects that need more than surface-level features: dynamic behavior, readable data structures, long-term maintainability, and tools that help complex communities actually run.

## Current Focus

* Developing tools and infrastructure for **AG-GS / Avant-Garde**, including live world-state tracking, national statistics, wiki systems, admin workflows, and simulation support.
* Building and improving **Arma 3** mission systems, AI behavior, logistics, faction configuration, and dynamic battlefield mechanics.
* Experimenting with **AI-assisted development workflows**, agent systems, and automation for complex project maintenance.
* Working on better documentation standards for large, evolving projects.

## Featured Work

### [AG-GS / Global Ledger](https://github.com/Azraeel/AG-GS)

A live reference and administration system for a persistent nation-roleplay world. Global Ledger supports national statistics, trade, population, military readiness, naval inventories, intelligence records, elections, world events, wiki content, admin tools, live updates, and public read-only browsing.

### [FLOPS_FLO.gc_krayina](https://github.com/Azraeel/FLOPS_FLO.gc_krayina)

An Arma 3 dynamic frontline operations mission focused on intelligent OPFOR behavior, logistics networks, garrison systems, intel mechanics, faction customization, dynamic vehicle spawning, and automated battlefield structure.

### [M28AI](https://github.com/Azraeel/M28AI)

Work connected to Arma 3 AI systems and battlefield behavior. This project fits into my broader interest in improving tactical simulation, unit behavior, mission flow, and large-scale scenario reliability.

### [NR6-HAL](https://github.com/Azraeel/NR6-HAL)

Repository support for tracking issues, documentation, and development around Hetman’s Artificial Leader systems for Arma 3.

## What I Care About

I like projects that survive contact with scale. Clean code matters, but so does clear organization, useful documentation, and making sure systems can still be understood six months later.

My general priorities are:

* Practical tools over flashy demos
* Simulation depth without unnecessary complexity
* Maintainable project structure
* Documentation that helps future contributors
* Systems that can be expanded without collapsing
* Real workflows for real communities

## Project Direction

My current work is increasingly focused on turning complex creative and simulation projects into usable infrastructure. That means building tools that make world states easier to manage, game systems easier to maintain, and large repositories easier to document and improve.

I am especially interested in:

* Persistent world management tools
* Dynamic military and strategy simulations
* AI-supported development pipelines
* Procedural and semi-procedural content systems
* Better tooling for roleplay, wargaming, and strategy communities
